# 云同步选择与本地 JSON 备份设计

**状态：** 已由用户于 2026-07-12 确认，等待实施计划评审。

## 目标

让用户能在“管理云同步”中选择参与云同步的数据类型，并提供始终覆盖全部七张业务表的本地明文 JSON 备份与安全恢复能力。同步选择必须同时约束手动上传、手动下载、所有 CRUD 自动推送、应用启动恢复和云端变更拉取。

## 已确认的范围与决策

### 数据表

云同步数据集固定为七张表，不新增第八张云表：

| 云表 | 中文名称 | 是否可由用户独立取消 |
| --- | --- | --- |
| `cryptoparams` | 加密参数 | 否；任一加密业务表被选中时自动包含 |
| `usersettings` | 个性化与协议设置 | 是 |
| `remotehosts` | 远程主机 | 是 |
| `rdpcredentials` | Windows 登录凭据 | 是 |
| `rustdeskrelays` | RustDesk 中继配置 | 是 |
| `sshkeys` | SSH 密钥 | 是 |
| `totpentries` | 双重验证动态口令 | 是 |

`cryptoparams` 不作为用户可关闭的独立业务选项。只要用户选择 `remotehosts`、`rdpcredentials`、`rustdeskrelays`、`sshkeys` 或 `totpentries` 中任一含敏感字段的表，同步执行集必须自动加上 `cryptoparams`，并在拉取时始终优先执行。`usersettings` 可以单独取消。

同步选择不写入云端，也不影响其他设备；它以本机 Preferences 保存，每台设备初始都默认选中全部七表。用户取消某表不会删除本机或云端已有内容，只会停止未来的传输。允许所有业务表均取消；此时上传和下载入口禁用，并明确说明“请先选择至少一类同步数据”。

### 设置页与 Sheet 交互

“云同步”顶层卡保持在“外观”与“连接实况窗”之间，视觉、圆角、毛玻璃、阴影、间距和展开/收起动画复用现有设置卡风格。展开后由三张同风格操作卡组成：

1. `管理云同步`：进入云同步根 Sheet；副标题显示已选择的业务类型数量，服务未就绪时显示华为云空间与系统同步开关提示。
2. `本地备份`：先在 Sheet 中展示敏感明文风险，再由系统文件保存器选择位置并生成 JSON。
3. `从本地恢复`：选择 JSON 后展示预检摘要与覆盖本机数据的二次确认；确认前不写数据库。

继续使用现有“设置叶子 Sheet 单根宿主 + 内部路由状态”架构，不能创建嵌套 `bindSheet` 宿主。云同步根 Sheet 的三项为“上传当前设备数据”“下载云端数据到本地”“管理同步数据表”。上传和下载保留现有二次确认及风险语义；确认页使用明显危险操作样式和返回入口，执行中锁定关闭及重复点击。系统云同步不可用时，任何云端操作都进入不可用页，明确要求用户确认：已登录同一华为账号、已开通华为云空间，并在系统设置打开云同步。

“管理同步数据表”是同一根 Sheet 的内部页面，显示七张中文说明卡。每张可独立选择的业务卡有勾选状态、用途说明和敏感性提示；`cryptoparams` 显示为“由加密数据自动保护”的依赖状态而非普通可勾选卡。状态改变立即写本机 Preferences；离开再进入能恢复选择。

### 云同步执行规则

现有方向规则不改变：启动恢复和手动下载使用 `SYNC_MODE_CLOUD_FIRST`；手动上传与 CRUD 自动推送使用 `SYNC_MODE_NATIVE_FIRST`。变更点是所有执行路径在调用 `cloudSync` 前必须通过同一个选择策略计算“有效表集合”。

| 触发路径 | 必须行为 |
| --- | --- |
| 手动上传 | 仅将有效表集合按本地优先顺序推送；不选中的表绝不调用云同步 API。 |
| 手动下载 | 仅拉取有效表；若有效集合包含加密业务数据，先拉取 `cryptoparams`。 |
| 应用启动恢复 | 仅拉取有效表；拉取后仅刷新受影响服务与偏好。 |
| 云端数据变更事件 | 仅拉取有效表；不能把未选表的云变更写进本机。 |
| 主机、凭据、中继、SSH 密钥、TOTP、可同步设置的 CRUD | 仅当所属表在有效集合中时，才进入现有按表串行/合并自动推送队列。 |

将选择策略设为唯一入口，禁止 UI、服务或 `CloudStore` 分别维护表判断。未选中的表仍保留本机 CRUD 与内存刷新；云服务未开启时也保留本地数据库功能。

### 本地 JSON 备份

本地备份无视同步勾选，始终导出七张表的原始 RDB 行数据。文件采用用户明确选择的明文 JSON 格式，保存前和恢复前都必须明确提示其中可能含 SSH 私钥、TOTP 密钥、远程登录密码、RustDesk 中继密钥和账户密码；应用不会把备份上传到云端，也不会替用户加密该文件。

文件包络固定如下：

```json
{
  "format": "remotedesktop-local-backup",
  "version": 1,
  "createdAt": 1783830000000,
  "appVersion": "1.0.5",
  "tables": {
    "cryptoparams": [],
    "usersettings": [],
    "remotehosts": [],
    "rdpcredentials": [],
    "rustdeskrelays": [],
    "sshkeys": [],
    "totpentries": []
  },
  "sha256": "hex digest of the canonical payload without sha256"
}
```

“canonical payload”是固定键顺序的 `format`、`version`、`createdAt`、`appVersion`、`tables` JSON 字符串；每张表按主键稳定排序，确保相同内容可复现校验。导出使用现有 `DocumentViewPicker.save()` 和 `fileIo` 模式，默认文件名为 `RemoteDesktop-backup-YYYYMMDD-HHmmss.json`。用户取消保存不改任何数据。

### 本地恢复

恢复流程依次为：选择 JSON → 只读读取 → 解析 → 校验包络格式、版本、七表完整性、各行允许字段和类型、SHA-256 → 展示备份创建时间/版本/各表条数 → 用户二次确认覆盖 → 单个 RDB 事务替换七表 → 刷新 `HostSyncService`、`KeyVaultService`、中继和可同步偏好内存态。

校验或文件读取失败时，不启动事务且不修改本机数据。事务中的任何错误必须回滚，原数据完全保留。恢复执行期间设置全局“抑制自动云推送”保护，避免表替换触发的 CRUD/观察链路反向覆盖云端；恢复完成后不自动上传，只有用户随后手动上传或进行新的本机 CRUD 才能把数据写回云端。恢复不受同步选择影响，始终恢复七表。

## 方案比较

1. **推荐：本机 Preferences 选择策略 + `CloudStore` 单一执行门控。** 不改变云 schema，不影响其他设备；所有同步路径使用同一有效表计算，最容易测试并避免漏网路径。
2. 将选择状态同步到云端。设备间可复用偏好，但一台设备取消同步会意外停止另一台设备的数据保护，不符合“每台设备独立”的已确认需求。
3. 为每张表分别在 UI 层拦截。改动表面较小，但自动同步、启动恢复和云事件容易漏改，且加密依赖会重复实现，因此不采用。

## 错误处理与可观测性

- 每个云操作返回已执行表、跳过表和失败表；UI 只在全部有效表成功后显示成功，部分失败要保留可读错误与重试入口。
- 云端不可用、零表选择、读取失败、哈希不匹配、格式版本不支持、字段不合法、事务失败均有明确用户提示，日志不得写出主机地址、账号、密码、私钥、TOTP 密钥或完整备份内容。
- 本地备份成功仅显示文件名和表计数，不能在日志或 toast 回显敏感 JSON。

## 测试与验收

- 纯策略测试：默认全选、独立取消、零表禁用、`cryptoparams` 自动依赖、稳定执行顺序、每个触发路径过滤。
- 备份策略测试：七表完整性、canonical payload、哈希一致/不一致、旧版/未知表/缺表/错误字段拒绝。
- RDB 服务测试：导出所有表、恢复成功替换、任一校验失败不写入、事务失败回滚、恢复期间不触发自动推送。
- UI 手工验收：卡片位置与折叠动画、三入口、表选择持久化、上传/下载二次确认、华为云空间不可用提示、备份明文风险提示、恢复预览及二次确认。
- 最终运行生产 `assembleHap`；`default@OhosTestBuildArkTS` 若继续被既有 HostListPage/SourceMap 故障阻断，记录完整报错并单独说明，不将其视为本功能失败。

## 非目标

- 不新增或迁移华为云表，不清理未同步的数据，不改变现有加密格式或 `usersettings` 白名单。
- 不实现备份文件加密、云端备份版本历史、自动定时备份、跨设备同步勾选或冲突合并策略。
- 不修复与本功能无关的既有 ohosTest/SourceMap 问题。
