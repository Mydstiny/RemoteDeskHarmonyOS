# 云同步条件说明与设置 Sheet 生命周期修复

**日期：** 2026-07-13
**范围：** 设置页云同步区域与本地恢复文件选择流程
**不变约束：** 不修改七张云表及字段；不修改 RDP、RustDesk、SSH、VNC、渲染、音频、输入或加密数据格式。

## 问题

1. 系统没有公开、可靠的 API 让应用直接读取“华为云空间是否开通”或“系统云同步开关是否开启”。系统开关关闭时，分布式表注册仍可能成功，因此不能承诺即时、精确地弹出“开关已关闭”提示。
2. 用户从云同步的“从本地恢复”启动系统文件选择器后，点击右上角系统叉号关闭设置 Sheet，后续设置内所有弹窗都无法打开；离开设置页的 FAB 弹窗不受影响。

## 根因

设置内二级弹窗共用 `showSettingsLeafSheet`。`openSettingsLeafSheet()` 在该状态为 `true` 时直接返回，避免重复挂载。系统叉号触发 `bindSheet.onDisappear`，但现有回调只记录关闭时间，未把 `showSettingsLeafSheet` 复位为 `false`。因此原生 Sheet 已关闭而 ArkTS 状态仍为打开，后续设置操作被错误拦截。FAB 使用独立的 `showAddSheet`，所以仍可正常工作。

## 设计

### 固定“云同步详情”卡片

在“云端数据”卡片组中，位于“管理云同步”之后、本地备份之前，新增始终可见的“云同步详情”行，沿用现有图标、圆角、背景、分隔线与触摸反馈。

卡片展示：

- 应用端当前状态：端云数据库已就绪，或“应用暂未建立端云连接”。
- 正常使用的四个前提：同一华为账号、已开通华为云空间、系统设置中已开启云同步、网络可用。
- 说明应用不能直接判定系统开关状态；连接异常时引导用户检查上述条件，而不把所有网络错误误报为“云服务未开通”。

点击卡片打开只读的云同步说明 Sheet，不触发上传、下载或任何本地/云端数据写入。

### 设置 Sheet 统一关闭与恢复选择交接

新增一个设置 Sheet 生命周期出口，负责：取消待执行的 Sheet 打开定时器、清空 `showSettingsLeafSheet`、记录关闭时间，并按需保留当前云同步步骤数据。

- 系统叉号、手势关闭、主动关闭和 `onDisappear` 都调用同一出口；重复调用必须安全。
- `onDisappear` 不再只是记时间，必须同步 ArkTS 可见状态，消除“原生已关闭、状态仍打开”的分裂。
- 从本地恢复改为先关闭当前云同步 Sheet，再启动系统文件选择器；不在同一 UI 时刻并发持有二级 Sheet 与系统 picker。
- picker 取消、无效文件或读取失败后，设置页保持可继续操作；不保留伪打开状态。
- picker 返回有效 JSON 后，重新通过既有延迟保护打开云同步 Sheet，并进入“确认本地恢复”。恢复成功或失败后同样走统一关闭/保留流程。

## 错误处理

- 系统云同步/云空间无法精确探测时，固定详情卡片承担常驻说明；已有的“服务不可用”错误引导仅在 API 返回可识别服务错误时展示。
- 文件选择取消和校验失败是用户可恢复状态：显示当前 Sheet 内的错误信息或返回操作入口，不锁定页面。
- 恢复数据库失败仍沿用现有事务回滚和本地恢复锁；本次 UI 修复不得改变恢复的云端不自动上传约束。

## 验证

新增纯策略测试覆盖：

1. 设置 Sheet 被外部关闭后，状态允许再次打开。
2. 关闭会取消未执行的延迟打开任务。
3. 本地恢复选择仅在拿到有效备份后请求“确认恢复”Sheet。
4. 云同步详情卡片的状态文案不会把“应用未连接”断言为“系统开关已关闭”。

真机回归：打开“从本地恢复”后点击系统右上角叉号；依次打开外观、云同步、加密等设置弹窗，均应正常。随后分别取消文件选择、选择无效 JSON、选择有效 JSON 并成功恢复，重复验证所有设置弹窗。

## 实施与验证记录（2026-07-13）

### 已实施范围

- `520862e4 feat(cloud): describe sync prerequisites` 新增 `CloudSyncGuidancePolicy` 与回归断言。无端云连接时文案固定为“应用暂未建立端云连接，请检查使用条件”；它只描述应用端连接状态，不把结果断言成系统云同步开关或华为云空间状态。
- `2a180bd9 fix(settings): define leaf sheet lifecycle` 新增纯生命周期策略：关闭后的叶 Sheet 状态固定为不可见、`mode=0`，并取消尚未执行的延迟打开；本地恢复确认仅接受已校验的备份文档。
- `dd6ffbbc fix(settings): recover sheet lifecycle after local restore` 将策略接入设置页。云端数据区域的“云同步详情”位于“管理云同步”和“本地备份”之间，始终展示，并可打开只读详情 Sheet 显示四项使用前提；该步骤没有上传、下载、恢复或写入操作。
- 设置叶 Sheet 的 `onDisappear` 现在区分外部/系统关闭与程序性关闭：外部关闭会完整复位 ArkTS 可见状态；程序性关闭会保存旧 mode 快照，使迟到的 `onDisappear` 不会取消紧随其后的新打开计时器或覆盖新 mode。加密弹窗原有的关闭后状态刷新仍保留。
- “从本地恢复”现在只启动文件选择器。只有 `selectAndValidate()` 返回有效 JSON 后，才设置待恢复文档并经既有延迟保护打开“确认本地恢复”Sheet；取消、无效文件或异常仅提示，不遗留伪打开的 Sheet 状态。恢复成功只执行本地恢复、刷新列表并提示“未自动上传到云端”，没有新增上传入口。
