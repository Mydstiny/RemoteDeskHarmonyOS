# 主机列表精简与更新说明布局设计

**日期：** 2026-07-12  
**状态：** 已确认，待实现  
**工作区：** `C:\Users\14288\DevEcoStudioProjects\RemoteDesktop`（分支 `codex/rdp-rustdesk-video-performance`）

## 目标

移除远程主机页面中没有实际价值的收藏、最近和名称排序能力；恢复历史版本的沉浸光感悬浮底栏；将版本更新 Swiper 在 Pad 和 PC 上呈现为居中的半屏弹窗，同时保持手机和首次安装教程的既有行为。

## 已比较的实现方式

1. **推荐：保留现有页面路由，以响应式容器实现更新说明弹窗。** 更新说明仍由 `GuidePage`、现有版本记录表和一次性偏好控制；仅在 Pad/PC 上将内容置入带遮罩的居中卡片。这样不改变首次启动、版本已读和“继续”退出流程。
2. 使用系统 `bindSheet` 承载更新说明。视觉接近弹窗，但当前更新流程本身是启动路由；额外增加 Sheet 的关闭、返回和版本已读时机，容易绕开必须点击“继续”的流程，因此不采用。
3. 仅缩小全屏页面中的内容区域。实现最少，但没有弹窗层级和背景遮罩，不满足 Pad/PC 上“弹窗出来”的观感，因此不采用。

## 设计

### 1. 移除收藏、最近与名称排序

- 主机列表不再显示“收藏”“最近”“名称”三个栏位，也不保留对应的页面状态、点击处理或排序策略。
- 删除收藏星标、收藏切换入口、收藏优先排序和服务层收藏操作；应用内不再读取、写入或展示主机收藏状态。
- 兼容已存储的旧主机数据：历史数据中的收藏属性在读取时不参与模型与显示逻辑，不触发数据库结构迁移，也不会再重新显示。
- 列表继续保留搜索、协议筛选、主机分组/工作区等与实际管理有关的能力；展示顺序使用主机服务的基础顺序，不提供无意义的客户端二次排序。
- 现有主机连接、编辑、删除、同步、密钥库和协议功能不改变。

### 2. 恢复沉浸光感悬浮底栏

- 以提交 `6628d6b3` 中已验证的 `HdsTabs` 悬浮栏配置为视觉基线，做定点恢复，而不是回退整个页面。
- 手机与 Pad 的底栏恢复以下体验：64vp 高度、280/320/380vp 自适应宽度、8vp 侧边距、按断点变化的底边距、92vp 光晕遮罩、`IMMERSIVE + ADAPTIVE` 材质以及握姿自适应。
- 保留当前主机、密钥库、设置三个 Tab 及其导航状态；不修改传感器生命周期、主机页面业务或设置页逻辑。
- PC 不新增握姿或移动端底栏行为，仍按现有桌面布局工作。

### 3. 响应式版本更新 Swiper

- **手机：** 维持当前全屏更新说明体验。
- **Pad 与 PC：** 仅当模式为“版本更新说明”时，在启动页面上使用半透明背景遮罩和居中卡片，形成弹窗效果。首次安装的“简单使用教程”继续全屏，不受影响。
- 弹窗卡片采用当前主题的系统材质、圆角、边框和阴影；宽度受屏幕限制（约 560–640vp），高度约占可用视口 56–62%，在较小 Pad 上保留安全边距。内容区继续使用同一 Swiper、页码点和底部按钮。
- 不提供点击遮罩或手势关闭。用户必须在最后一页点击“继续”，以保持“新版本首次打开只展示一次”的版本已读语义。
- 版本记录、页面文案和 `last_seen_release_version` 的持久化逻辑不改变；最后一页继续显示“欢迎使用 1.0.6”和“继续”。

## 架构边界

| 单元 | 职责 | 不负责 |
| --- | --- | --- |
| `HostListPage` | 移除无用的收藏/排序控件；恢复既有 HdsTabs 外观 | 修改主机连接或同步流程 |
| `HostWorkspacePolicy` 与主机模型/服务 | 删除收藏排序与写入接口，兼容遗留数据 | 改变分组和搜索规则 |
| `GuidePage` | 根据设备断点选择全屏或弹窗式内容容器 | 更改首次启动与版本已读决策 |
| `ReleaseNotesPolicy` | 提供可测试的“仅 Pad/PC 更新说明使用弹窗”判定 | 存储或路由实现 |

## 验收标准

1. 主机页面不再有收藏、最近、名称栏位、星标或收藏操作；原有主机仍可搜索、分组、连接、编辑与删除。
2. 手机/Pad 的底栏恢复沉浸材质与光晕悬浮效果，Tab 切换与握姿自适应不退化；PC 不出现移动端底栏副作用。
3. 手机首次安装教程和更新说明仍全屏；Pad/PC 的更新说明为半屏居中弹窗，教程仍全屏。
4. Pad/PC 更新说明的遮罩不可误关闭；最后一页“继续”后进入原有登录流程，并且同一版本不再重复展示。
5. 为删除的收藏排序、更新说明呈现策略和关键响应式判定新增或更新纯 ArkTS 测试；生产 `assembleHap` 通过，并对改动范围运行静态检查。

- 不进行旧主机持久化数据的破坏性迁移。
- 不改变首次安装教程的全屏交互。
